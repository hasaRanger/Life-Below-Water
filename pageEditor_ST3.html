Feedback Page

Technical Description

The Feedback Page is implemented using a combination of HTML, CSS and JavaScript to ensure a structured, user friendly and interactive interface. Below are the key techniques and choices used in its development.

HTML Structure

  • Semantic Tags: This page is structured with <header>, <nav>, <section> and <footer> to enhance readability, SEO and accessibility.
  • Forms and Inputs: The feeddback form is created using <form> and various input elements such as:
    • <input type="text">, <input type="email"> and <input type="tel"> for user details.
    • <textarea> for additonal comments.
    • <input type="radio"> and <input type="checkbox"> for surveying and multiple-choice feedback.
  • Tables: Tables (<table>, <tr>, <th> and <td>) are used for structred data display and feedback collection.
  • Navigation: A collapsible menu is implemented using JavaScript for better mobile usability.
  • Validation Attributes: Attributes like required and maxlength are used to enforce user input constraints wihtout relying solely on JavaScript.

CSS Styling

  • Background and Gradient Effects:
    • The page background uses background: linear-gradient(to bottom, #E0E0E0, #005F73); for a smooth color transition.
    • Input fields have a gradient effect with background: linear-gradient(to right, white, #94D2B2); .
  • Typography and Spacing:
    • font-family: Arial, sans-serif; for ensures readability.
    • Diffferent fonts like 'Courier's New', 'Cambria' and Georgia are used for various sections.
  • Responsiveness;
    • margin-left: auto; margin-right: auto; centers elements dynamically.
  • Animations and Tranisitions;
    • table-container and .otherTextArea use transition: max-height 0.5s ease-in-out, opacity 0.5s ease-in-out; for smooth content expansion.
    • Button hover effects are achieved using transition: 0.2s ease-in; .
  • Custom Radio and Checkbox Styling:
    • appeareance: none; removes default styles , allowing for custom design.
    • border: 1px solid black; border-radius: 50%; provides a circular shape.
    • input [type="radio"]:checked:before sets a filled indicator for selected radio buttons.

JavaScript Functionalities

  • Dropdown Toggle for Subscription Preference:
    • toggleDropdown() dynamically shows the subscription question dropdown when "Yes" is selected and hides when "No" is selected.
  • Expanding Additional Questions:
    • toggleUserReturnTable() displays a hidden feedback table when users indicate they have previously visited the website.
    • toggleOtherTextArea() reveals a text input field when the "Other" checkbox is selected.
  • Event Listeners and Transitions:
    • JavaScript smoothly animates visibility changes by modifying max-height and opacity properties is CSS.

Accessibility

Ensuring accessibility is crucial for making the Feeback Page usable by people with disabilities and improving overall user experience. Below are the key accesssiblity considerations implemented.

  • <label> elements are explicitly linked to form controls via for attribute to help user who rely on screen readers.
  • The collapsible menu .dropdown respond to keyboard navigation for improved usability.
  • The background uses high-contrast color gradient(background: linear-gradient(to bottom, #E0E0E0, #005F73)) to improve readibility.
  • Sufficient contrast between text and background to meet WCAG (Web Content Accessibility Guidelines) Standards.
  • Font choices (Arial, Cambria, Georgia) ensures legibilty over variety of devices.
  • Form fields include clear labels and contextual hints for users with cognitive disablilites.
  • Animations and transitions are kept subtle to avoid triggering motion sickness for sensitive users.

Team Page

Technical Description

Unique HTML and CSS features have been utilized here to implement the Team Page.

HTML Structure

  • Semantic Tags: This page is structured with <header>, <nav>, <section> and <footer>, enhancing readibility and accessibility.
  • .image-container: This division block serves as a flexible container that holds all the team member cards(image-box) and arranges them neatly.
  • .image-box: This block holds a portrait image for each team member and a description box with information of relevant member.
  • .caption: Description box for each team member is held by this block with information such as name, link to the Page Editor and social media handles.
  • Navigation: A collapsible menu is implemented using JavaScript for better mobile usability.

CSS Styling

  • Background and Gradient effects:
    • Background of the page has background: linear-gradient(to bottom, #E0E0E0, #005F73); for a smooth color transition.
  • Typography and Spacing:
    • Arial, Helvetica, sans-serif; and Verdana, Geneva, Tahoma, sans-serif; are used as font styles to gain user traction.
  • Responsiveness:
    • display: flex; is used in .image-container to arrange team memnbers dynamically.
    • flex-wrap: wrap; ensures that items move to the next row on smaller screen instead of unnecessary shrinking.
    • justify-content:center; ensures that .image-box elements are evenly distributed across different screen sizes.
    • overflow-x: hidden is applied to the body which hides horizontal scrolling throughout the entire webpage, preventing altercations with felxboxes and other animations.
    • gap: 20px; provides consistent spacing between .image-box elements without relying on fixed margins.
    • margin: 0 10px prevents elements from sticking to the edges of the screen.
  • Animations and Transitions:
    • Images inside .image-box gets it's opacity increased when cursor is hovered over each team card with transform: scale(1.1); smoothly.
    • .caption is hidden off-screen by default, on hover, it slides in from the left left: 0; of the team card and slides back into the left of the team card as the cursor moved away.
    • Transition of .caption starts 0.3s after the transtion of the image in .image-box preventing overlaps between transitions.
    • Social media logos get smooth enlargement when they get hovered over with, and return to the original size as soon as the cursor moves away them.

Accessibility

To create an inclusive experience, accessibility plays a crucial role in making the Team Page usable for people with disabilities and improving usability for all. Below are the key accessibility enhancements implemented.

  • <header>, <nav>, <section> and <footer> improve document structure for screen readers.
  • Proper use of <h1> tag for main topic.
  • <img> elements include alt attributes, ensuring visually impaired users can understant image content.
  • Flexbox-based layout adjusts dynamically for different screen sizes.
  • Background gradient and Fonts like Arial, Helvetica meets WCAG (Web Content Accessibility Guidelines) Standards.
  • Team member names and IDs are inside structured <div> containers.
  • Smooth transitions (transition: ease-in-out;) preventing overwhelming animations.

Content Page

Technical Description

This content page presents a detailed analysis about Threats towards marine life which is loaded with pictures and facts, facilitated with many HTML and CSS implementations.

HTML Structure

  • Semantic tags: This page is structured with <header>, <nav>, <div> and <footer>, enhancing usability and readibility.
  • .threat-container: This block serves as a flexible container that holds cards(.threat-left, .threat-right) that contains certain facts related to the topic.
  • .threat-left & .threat.right: Orders of how the subtopic, description and the image positioned inside the card is switching with the each card, improving positive user experience.

CSS Styling

  • Background and Gradient effects:
    • A gradient of background: linear-gradient(to bottom, #E0E0E0, #005F73) is used in the background for a smooth color transition.
    • .introduction contains background:rgba(148, 210, 178, 0.3); which matches with the background colors.
    • Each .threat-left & .threat-right got a gradient background: linear-gradient(to bottom, white, #94D2B2); emphasizing them well, out of the background.
  • Typography and Spacing:
    • .introduction uses 'Lucida Sans', 'Lucida Sans Regular', 'Lucida Grande', 'Lucida Sans Unicode', Geneva, Verdana, sans-serif;, which is a rounded and reader-friendly font.
    • Sub-headings such as .threat-left-topic & .threat-right.topic has 'Bebas Neue', sans-serif;, giving them a bold and modern look.
    • Descriptive text (.threat-left & .threat-right) got 'Oswald', sans-serif;, creating strong visual impact by empahsizing key informations.
    • padding-bottom:5px; between list items improves readibility by preventing text from being too crammed.
    • .threat-left & .threat.right have margin.bottom: 100px; and .threat-caption-left, threat-caption-right have margin-;eft: 40px; margin-right: 40px; avoiding visual clutter, making it easy to differentiate between different threats.
  • Responsiveness:
    • .heading, .introduction, .threat-container are positioned at the center of the page with align-items: center;, for better readibility and usability.
    • A Go-to-top button is added to the page for easy travellling back to the start of the page.
    • Special terms inside the descriptive text (.benthic, .PAC) contains brief explainations included within a box, which would be appeared when the cursor is hovered over the relevant word.

Accessibility

A user-friendly Content Page should accomodate all individuals, regardless of ability. The adjustments listed below have been made to support a more seamless interactions for everyone.

  • The page has properly used semantic HTML elements such as <header>, <nav>, <section> and <footer>, ensuring better screen reader compatibilty.
  • <img> elements include <alt> attribute assisting visually impaired users.
  • body text has got color: #00364B which suit with the color tone in the background.
  • Inside descriptive text there are three types of highlighted data available, with a speciic color codings scheme.
    • Quantitative Data (.highlight1) : color: #00364B which represents legitimacy and accuracy, aligning with the precision required for numerical data.
    • Categorical Data (.highlight2) : color: #043E72 which differentiates categories from numbers, while still maintaining a professional and readable tone.
    • Temporal Data (.highlight3) : color: #050724 which is an appropriate color for time-based trends.

Challenges and Lessons Learned

  • Formatting the user experience forum inside the feedback form, which seeks for user ratings on the overall experience of the webpage in a well-organized manner.
    • Used <table>, first row with <th> having value of rating options and first column having elements inside the webpage which need to be rated.
    • Under each rating option, <input type="radio"> used for user to select each option.
  • Asking the user for the reason to return to the website only if he visisted the website not for the first time.
    • .userReturn block appears only when the No radio button of .userVisit is selected with JavaScript in a smooth transition.
  • Making the webpages responsive to various device-viewports.
    • Using Media Queries (@media screen), elements inside each page are customized to adapt to different viewports.

Compliance

  • Accessibility Compliance: The website uses proper HTML semantics (<header>, <nav>, <section>, <footer>) and alt attributes for images, making it readable for screen readers.
  • Data Pretection: No personal data is stored or shared without consent.
  • Security Measures: The website does not include harmful scripts, SQL injection vulnerabilities, or unsafe user input handling.
  • Content Compliance: All media used (images, logos) are either original or sourced from royalty-free repositories, ensuring copyright laws are respected.
